    В этом разделе приведено описание формата, используемого в SQL ServerSQL Server для носителей резервных копий, соответствия между носителями резервных копий и устройствами резервного копирования, организации резервных копий на носителях резервных копий, а также даны некоторые соображения, касающиеся наборов носителей и семейств носителей.This topic describes the format that SQL ServerSQL Server uses for backup media, the correspondence between backup media and backup devices, the organization of backups on backup media, and several considerations for media sets and media families. Кроме того, в этом разделе описаны шаги инициализации или форматирования носителей резервных копий перед их использованием впервые или использованием для замены старого набора носителей новым набором носителей, а также показано, как перезаписать старые резервные наборы в наборе носителей и как добавить новые резервные наборы к набору носителей.The topic also describes the steps initializing or formatting backup media before you use it for the first time or replace an old media set with a new media set, how to overwrite old backup sets in a media set, and how to append new backup sets to a media set.

    Термины Terms

    набор носителейmedia set
    Упорядоченный набор носителей данных резервной копии в виде определенного количества ленточных устройств или дисков, на которые может быть записана одна или несколько операций резервного копирования, с использованием фиксированного типа и номера устройств резервного копирования.An ordered collection of backup media, tapes or disk files, to which one or more backup operations have written using a fixed type and number of backup devices.

    семейство носителейmedia family
    Резервные копии, созданные на одном устройстве без зеркального отображения или на наборе устройств с зеркальным отображением в наборе носителейBackups created on a single nonmirrored device or a set of mirrored devices in a media set

    резервный набор данныхbackup set
    Содержимое резервной копии добавляется на набор носителей при успешной операции резервного копирования.The backup content that is added to a media set by a successful backup operation.

    Общие сведения о наборах носителей, семействах носителей и резервных наборах Overview of media sets, media families, and backup sets

    Резервные копии на наборе из одного или более носителей резервных копий образуют отдельный набор носителей.The backups on a set of one or more backup media compose a single media set. Набор носителей — упорядоченная коллекция носителей резервных копий, лент или дисковых файлов либо больших двоичных объектов Azure, на которые производили запись одна или несколько операций резервного копирования, получая доступ по фиксированному типу и номеру устройства.A media set is an ordered collection of backup media, tapes or disk files, or Azure Blobs, to which one or more backup operations have written using a fixed type and number of backup devices. Данный набор носителей использует накопители на магнитной ленте или дисковые накопители либо большие двоичные объекты Azure, но не сочетание двух или нескольких типов носителей.A given media set uses tape drives, or disk drives or Azure blobs, but not a combination of two or more.

    Например , устройства резервного копирования, объединенные в набор носителей, могут включать в себя три ленточных накопителя с именами \\.\TAPE0, \\.\TAPE1и \\.\TAPE2.Example: Backup devices associated with a media set might be three tape drives named \\.\TAPE0, \\.\TAPE1, and \\.\TAPE2. Этот набор содержит только ленточные носители, имеющие не менее трех лент (по одной на ленточный накопитель).That media set contains only tapes, starting with a minimum of three tapes (one per drive). Тип и номер устройства резервного копирования присваиваются при создании набора носителей и не могут меняться.The type and number of backup devices are established when a media set is created, and they cannot be changed. Однако при необходимости, между операциями резервного копирования и восстановления можно заменить устройство на другое устройство аналогичного типа.However, if necessary, between backup and restore operations a given device can be replaced with a device of the same type.

    Набор носителей создается на носителе резервной копии во время операции резервного копирования при форматировании носителя.A media set is created on the backup media during a backup operation by formatting the backup media. Дополнительные сведения см. в подразделе Создание нового набора носителейниже в данном разделе.For more information, see Creating a New Media Set, later in this topic. После форматирования каждый файл или лента содержат заголовок носителя и готовы к записи данных.After formatting, each file or tape contains a media header for the media set and is ready to receive backup content. После формирования заголовка операция резервного копирования продолжает создавать резервную копию указанных данных на все заданные устройства резервного копирования.With the header in place, the backup operation proceeds to back up the specified data to the backup media on all of the backup devices specified for the operation.

    ПРИМЕЧАНИЕ.NOTE! Для предотвращения потери данных в случае повреждения носителя (ленты или файла на диске), наборы носителей можно сделать зеркальными.Media sets can be mirrored to protect against a damaged media volume (a tape or disk file). Дополнительные сведения см. в подразделе Зеркальные наборы носителей резервных копий (SQL Server).For more information, see Mirrored Backup Media Sets (SQL Server).

    Сжатые и несжатые резервные копии не могут совместно использоваться в одном наборе носителей.Compressed and uncompressed backups cannot occur together in a media set. Считывание сжатых резервных копий возможно в любом выпуске SQL Server 2008SQL Server 2008 или более поздней версии.Any edition of SQL Server 2008SQL Server 2008 or later can read compressed backups. Дополнительные сведения см. в разделе Сжатие резервных копий (SQL Server).For more information, see Backup Compression (SQL Server).

    Семейства носителейMedia Families

    Резервные копии, созданные на отдельном устройстве, не имеющем зеркального отображения, или на наборе устройств с зеркальным копированием, принадлежащем одному набору носителей, образуют семейство носителей.Backups created on a single nonmirrored device or a set of mirrored devices in a media set constitute a media family. Количество устройств резервного копирования, используемых в наборе носителей, определяет количество семейств носителей в наборе носителей.The number of backup devices used for the media set determines the number of media families in a media set. Например, если набор носителей использует два незеркальных устройства резервного копирования, этот набор носителей содержит два семейства носителей.For example, if a media set uses two nonmirrored backup devices, the media set contains two media families.

    В зеркальном наборе носителей каждое семейство носителей является зеркальным.In a mirrored media set, each media family is mirrored. Например, если шесть устройств резервного копирования формируют набор носителей, в котором используются два зеркала, этот набор содержит три семейства носителей, каждое из которых содержит по две резервных копии данных.For example, if six backup devices are used to format a media set, where two mirrors are used, there are three media families, each containing two equivalent copies of backup data. Дополнительные сведения о зеркальных наборах носителей резервных копий см. в разделе Зеркальные наборы носителей резервных копий (SQL Server).For more information about mirrored media sets, see Mirrored Backup Media Sets (SQL Server).

    Каждой ленте или диску в семействе носителей присваивается порядковый номер носителя.Each tape or disk in a media family is assigned a media sequence number. Порядковый номер носителя для диска всегда равен 1.The media sequence number of a disk is always 1. Для семейства ленточных носителей порядковый номер исходной ленты равен 1, вторая лента получает номер 2 и т. д.In a tape media family, the sequence number of the initial tape is 1, the sequence number of the second tape is 2, and so forth. Дополнительные сведения см. в подразделе Наборы носителей, семейства носителей и резервные наборы данных (SQL Server).For more information, see Media Sets, Media Families, and Backup Sets (SQL Server).

    Заголовок носителяThe media header

    Каждый том носителя резервной копии (дисковый файл или лента) содержит заголовок носителя, который создается первой операцией резервного копирования, использующей эту ленту или диск.Every volume of backup media (disk file or tape) contains a media header that is created when by the first backup operation that uses the tape (or disk). Заголовок остается без изменений, пока носитель не будет отформатирован повторно.That header remains intact until the media is reformatted.

    Заголовок носителя содержит все сведения, необходимые, чтобы идентифицировать носитель (файл или ленту) и его место в семействе носителей.The media header contains all of the information required to identify the media (disk file or tape) and its place within the media family to which it belongs. Эти сведения включают в себя следующее.This information includes:

    • Имя носителя.The name of the media.

      Имя носителя не является обязательным, но рекомендуется постоянно использовать имена носителей, чтобы однозначно их различать.The media name is optionally, but we recommend consistently using media names that clearly identify your media. Имя присваивается носителю во время форматирования.A media name is assigned by whoever formats the media.

    • Уникальный идентификатор набора носителей.The unique identification number of the media set.

    • Количество семейств носителей в этом наборе носителей.The number of media families in the media set.

    • Порядковый номер семейства носителей, содержащего данный носитель.The sequence number of the media family containing this media.

    • Уникальный идентификатор семейства носителей.The unique identification number for the media family.

    • Порядковый номер носителя в семействе носителей.The sequence number of this media in the media family. Для дискового файла это значение всегда равно 1.For a disk file, this value is always 1.

    • Содержит ли описание этого носителя метку MTF или описание носителя.Whether the media description contains an MTF media label or a media description.

      ПРИМЕЧАНИЕ.NOTE! Во всех носителях, использующихся для операций резервного копирования или восстановления, применяется стандартный формат резервных копий под названием MicrosoftMicrosoft Tape Format (MTF).All media that is used for a backup or restore operation use a standard backup format called MicrosoftMicrosoft Tape Format (MTF). MTF позволяет пользователям задавать метку ленты, которая содержит описание, характерное для MTF.MTF allows users to specify a tape label that contains a MTF-specific description. SQL ServerSQL Server сохраняет любую метку носителя MTF, записанную другим приложением, но не записывает метки носителей MTF. preserves any MTF media label written by another application but does not write MTF media labels.

    • Метка носителя в формате MicrosoftMicrosoft Tape Format или описание носителя (в свободной форме).The MicrosoftMicrosoft Tape Format media label or the media description (in free-form text).

    • Название программы резервного копирования, записавшей метку.The name of the backup software that wrote the label.

    • Уникальный идентификатор поставщика программного обеспечения, с помощью которого отформатирован носитель.The unique vendor identification number of the software vendor that formatted the media.

    • Дата и время записи метки.The date and time the label was written.

    • Количество зеркал в наборе (от 1 до 4); 1 является признаком того, что устройство не является зеркальным.The number of mirrors in the set (1-4); 1 indicates an unmirrored device.

      SQL Server 2017SQL Server 2017 позволяет обрабатывать носители, отформатированные в более ранних версиях SQL ServerSQL Server. can process media formatted by earlier versions of SQL ServerSQL Server.

    Резервные наборы данныхBackup sets

    При успешном завершении операции резервного копирования к набору носителей добавляется один резервный набор данных .A successful backup operation adds a single backup set to the media set. Резервный набор данных описывается в терминах набора носителей, на котором создан этот резервный набор.The backup set is described in terms of the media set to which the backup belongs. Если носитель резервной копии состоит только из одного семейства носителей, это семейство содержит весь резервный набор данных.If the backup media consists of only one media family, that family contains the entire backup set. Если носитель резервной копии состоит из нескольких семейств носителей, резервный набор данных распределяется между ними.If the backup media consists of multiple media families, the backup set is distributed among them. На каждом носителе резервный набор данных содержит заголовок со своим описанием.On each medium, the backup set contains a header that describes the backup set.

    В этом примере показана инструкция Transact-SQLTransact-SQL , с помощью которой создается набор носителей с именем MyAdvWorks_MediaSet_1 для базы данных AdventureWorks2012AdventureWorks2012 , использующий три накопителя на магнитных лентах в качестве устройств резервного копирования:This example shows a Transact-SQLTransact-SQL statement that creates a media set called MyAdvWorks_MediaSet_1 for the AdventureWorks2012AdventureWorks2012 database using three tape drives as backup devices: 

    BACKUP DATABASE AdventureWorks2012  
TO TAPE = '\\.\tape0', TAPE = '\\.\tape1', TAPE = '\\.\tape2'  
WITH   
   FORMAT,  
   MEDIANAME = 'MyAdvWorks_MediaSet_1'

    В случае успешного выполнения данной операции резервного копирования будет создан новый набор носителей, содержащий заголовок, и один резервный набор данных, распределенный на трех магнитных лентах.If successful, this backup operation results in a new media set containing a new media header and one backup set spread across three tapes.     Как правило, после создания набора носителей все последующие операции резервного копирования добавляют резервные наборы данных к этому набору носителей.Typically, after a media set is created, subsequent backup operations, one after another, append their backup sets to the media set. Все носители, используемые резервным набором данных, составляют набор носителей независимо от числа вовлеченных в операцию носителей или устройств резервного копирования.All of the media used by a backup set make up the media set, regardless of the number of media or backup devices involved. Резервные наборы данных последовательно нумеруются согласно их позиции в наборе носителей, что позволяет указывать резервный набор для восстановления.Backup sets are sequentially numbered by their position in the media set, allowing you to specify which backup set to restore.

    Каждая последующая операция резервного копирования требует одно и то же количество и тип резервных устройств.Every backup operation to a media set must write to the same number and type of backup devices. Содержимое каждой последующей резервной копии распределяется между носителями всех используемых устройств резервного копирования.With multiple devices, as with the first backup set, the content of every subsequent backup set is distributed among the backup media on all of the devices. Продолжая приведенный выше пример, при проведении второй операции резервного копирования (разностное резервное копирование) данные добавляются на тот же набор носителей:To continue the above example, a second backup operation (a differential backup) appends information to the same media set: 

    BACKUP DATABASE AdventureWorks2012  
TO TAPE = '\\.\tape0', TAPE = '\\.\tape1', TAPE = '\\.\tape2'  
WITH   
   NOINIT,  
   MEDIANAME = 'AdventureWorksMediaSet1',  
   DIFFERENTIAL

    При восстановлении резервных копий можно использовать параметр FILE, чтобы указать, какие резервные копии использовать.When you are restoring backups, you can use you the FILE option to specify which backups you want to use. В следующем примере показано, как использовать предложения FILE =номер_файла_резервного_набора при восстановлении полной резервной копии базы данных AdventureWorks2012AdventureWorks2012 , за которой следует разностная резервная копия на том же наборе носителей.The following example shows the use of FILE =backup_set_file_number clauses when restoring a full database backup of the AdventureWorks2012AdventureWorks2012 database followed by a differential database backup on the same media set. Этот набор содержит три ленточных носителя, находящихся в накопителях на магнитной ленте \\.\tape0, tape1и tape2.The media set uses three backup tapes, which are on tape drives \\.\tape0, tape1, and tape2. 

    RESTORE DATABASE AdventureWorks2012 FROM TAPE = '\\.\tape0', TAPE = '\\.\tape1', TAPE = '\\.\tape2'

   RECOVERY;

    Creating a new media set Creating a new media set

    Для создания нового набора носителей необходимо форматировать носитель резервной копии (одну или несколько магнитных лент или дисковых файлов).To create a new media set, you must format the backup media (one or more tapes or disk files). В процессе форматирования резервный носитель изменяется следующим образом.The formatting process changes the backup media as follows:

    1. Удаляется старый заголовок (если он есть), фактически удаляется прежнее содержимое носителя резервной копии.Deletes the old header (if any), effectively deleting the previous contents of the backup media.

      При форматировании ленточного устройства удаляется все прежнее содержимое установленной на тот момент магнитной ленты.Formatting a tape device deletes all previous contents of the currently mounted tape. Форматирование диска затрагивает только файл, указанный для операции резервного копирования.Formatting a disk affects only the file that you specify for the backup operation

    2. Записывается новый заголовок носителя на носитель резервной копии (магнитную ленту или дисковый файл) в каждом из устройств резервного копирования.Writes a new media header on the backup media (tape or disk file) on each of the backup devices.

    Создание резервной копии на существующем наборе носителей Backing up to an existing media set

    При создании резервной копии на существующем наборе носителей имеются следующие два параметра:When you are backing up to an existing media set, you have the following two options:

    • Добавление к существующему резервному набору данных.Append to the existing backup set.

      Чтобы использовать доступное пространство наилучшим образом, новые резервные наборы данных обычно добавляются к существующим наборам носителей.To make the best possible use of the available space, new backup sets typically are appended to existing media set. Добавление к резервной копии сохраняет любые предшествующие резервные копии.Appending to the backup preserves any prior backups. Дополнительные сведения см. в подразделе Добавление к существующим резервным наборамниже в этом разделе.For more information, see Appending to Existing Backup Sets, later in this section.

    Добавление, которое является поведением по умолчанию инструкции BACKUP, можно явно указать с помощью параметра NOINIT.Appending, which is the default behavior of the BACKUP, can be explicitly specified by using the NOINIT option.

    • Перезапись всех существующих резервных наборов данных текущими копиями с оставлением на месте текущего заголовка носителя.Overwrite all existing backup sets with the current backup, leaving the current media header in place.

      SQL ServerSQL Server имеет защиту от случайной перезаписи носителя. backup has safeguards to prevent you from accidentally overwriting media. Однако резервная копия может автоматически переписать резервные наборы данных, которые достигли окончания предопределенного срока хранения.However, backup can automatically overwrite backup sets that have reached a predefined expiration date.

      При копировании на ленту может иметь смысл оставлять заголовки лент.For tape headers, leaving the header in place can make sense. Дополнительные сведения см. в подразделе Перезапись резервных наборовниже в данном разделе.For more information, see Overwriting Backup Sets, later in this section.

      Перезапись существующих резервных наборов данных задается с помощью параметра INIT инструкции BACKUP.Overwriting existing backup sets is specified by using the INIT option of the BACKUP statement.

    Appending to existing backup sets Appending to existing backup sets

    Резервные копии одной или разных баз данных в разные моменты времени могут храниться на одном носителе.Backups performed at different times from the same or different databases can be stored on the same media. При присоединении еще одного резервного набора данных к существующим копиям на носителе предыдущее содержимое носителя остается неизменным, и новая резервная копия записывается после окончания последней.By appending another backup set to existing media, the previous contents of the media remain intact, and the new backup is written after the end of the last backup on the media.

    По умолчанию SQL ServerSQL Server всегда присоединяет новую резервную копию к содержимому носителя.By default, SQL ServerSQL Server always appends new backups to media. Присоединение может выполняться только в конец носителя.Appending can occur only at the end of the media. Например, если том содержит пять наборов резервных копий, нельзя пропустить первые три и перезаписать четвертый новым резервным набором.For example, if a media volume contains five backup sets, it is not possible to skip the first three backup sets to overwrite the fourth backup set with a new backup set.

    Если для записи резервной копии на ленту используется инструкция BACKUP WITH NOREWIND, то в конце операции лента остается открытой.If you use BACKUP WITH NOREWIND for a tape backup, the tape will be left open at the end of the operation. Это позволяет записывать последующие резервные копии на ленту без ее перемотки и последующего поиска последнего резервного набора.This allows you to append further backups to the tape without rewinding the tape and then scanning forward again to find the last backup set. Список открытых накопителей на магнитной ленте можно получить из динамического административного представления sys.dm_io_backup_tapes. Дополнительные сведения см. в разделе sys.dm_io_backup_tapes (Transact-SQL).You can find the list of open tape drives in the sys.dm_io_backup_tapes dynamic management view; for more information, see sys.dm_io_backup_tapes (Transact-SQL).

    Для резервных копий Microsoft Windows и резервных копий SQL ServerSQL Server могут совместно использоваться одни и те же носители, но сами эти копии не являются функционально совместимыми.Microsoft Windows backups and SQL ServerSQL Server backups can share the same media, but they are not interoperable. SQL ServerSQL Server не позволяет создать резервную копию данных Windows. backup cannot back up Windows data.

    ВАЖНО!IMPORTANT! Сжатые и несжатые резервные копии не могут совместно использоваться в одном наборе носителей.Compressed and uncompressed backups cannot occur together in a media set. Любой выпуск SQL Server 2008SQL Server 2008 или более поздних версий способен считывать сжатые резервные копии.Any edition of SQL Server 2008SQL Server 2008 or later versions can read compressed backups. Дополнительные сведения см. в разделе Сжатие резервных копий (SQL Server).For more information, see Backup Compression (SQL Server).

    Overwriting backup sets Overwriting backup sets

    Перезапись существующих резервных наборов данных задается с помощью параметра INIT инструкции BACKUP.Overwriting existing backup sets is specified by using the INIT option of the BACKUP statement. Этот параметр производит перезапись всех резервных наборов данных на носителе. Сохраняется только заголовок носителя.This option overwrites all the backup sets on the media and preserve the media header, if any. Если у носителя нет заголовка, он будет создан.If no media header exists, one is created.

    При копировании на ленту может иметь смысл оставлять заголовки лент.For tape headers, leaving the header in place can make sense. При копировании на дисковые носители переписываются только файлы, используемые устройствами резервного копирования, которые указаны в операции резервного копирования; остальные файлы на дисках сохраняются неизменными.For disk backup media, only the files used by the backup devices specified in the backup operation are overwritten; other files on the disk are unaffected. При перезаписи резервных копий сохраняется каждый существующий заголовок носителя, и новая резервная копия создается в качестве первой для устройства.When overwriting backups, any existing media header is preserved, and the new backup is created as the first backup on the backup device. Если никакого заголовка у носителя нет, автоматически будет записан правильный заголовок, связанный с именем носителя и его описанием.If there is no existing media header, a valid media header with an associated media name and media description is written automatically. Если существующий заголовок носителя недопустим, операция резервного копирования завершается.If the existing media header is invalid, the backup operation terminates. Если носитель пуст, формируется новый заголовок носителя с указанными параметрами MEDIANAME, MEDIAPASSWORD и MEDIADESCRIPTION, если таковые имеются.If the media is empty, the new media header is generated with the given MEDIANAME, MEDIAPASSWORD, and MEDIADESCRIPTION, if any.

    Носители резервных копий не перезаписываются, если имеет место любое из следующих условий:Backup media is not overwritten if either of the following conditions exists:

    • Срок действия резервных копий на носителе еще не истек.The existing backups on the media have not expired. (При указании аргумента SKIP срок действия не проверяется.)(If SKIP is specified, expiration is not checked.)

      Дата истечения срока хранения указывает день, когда истечет срок хранения резервной копии, и она сможет быть заменена другой копией.The expiration date specifies the date that the backup expires and can be overwritten by another backup. Дату истечения срока хранения резервной копии можно указать при ее создании.You can specify the expiration date when a backup is created. По умолчанию дата истечения срока хранения определяется набором параметров media retention хранимой процедуры sp_configure.By default, the expiration date is determined by the media retention option set with sp_configure. Дополнительные сведения см. в подразделе sp_configure (Transact-SQL).For more information, see sp_configure (Transact-SQL).

    • Имя носителя, если оно указано, не соответствует имени на носителе резервной копии.The media name, if provided, does not match the name on the backup media.

      Имя носителя представляет собой описательное имя, которое используется, чтобы облегчить распознавание носителя.The media name is a descriptive name used for easy identification of the media.

      Если существует уверенность в том, что необходимо перезаписать существующий носитель (например, если известно, что резервные копии на ленте более не нужны), можно в явном виде отказаться от этих проверок.If you are sure you want to overwrite the existing media (for example, if you know that the backups on the tape are no longer needed), you can explicitly skip these checks.

      Если носитель резервной копии защищен паролем Microsoft Windows, Microsoft SQL Server не будет производить записи на этот носитель.If the backup media is password protected by Microsoft Windows, Microsoft SQL Server does not write to the media. Чтобы перезаписать защищенный паролем носитель, нужно его заново инициализировать.To overwrite media that is password protected, you must reinitialize the media.

    Порядковые номера Sequence numbers

    Для нескольких семейств носителей в наборе носителей или нескольких резервных носителей в семействе носителей важен правильный порядок.The correct order is important for multiple media families within a media set or multiple backup media within a media family. Поэтому резервное копирование назначает порядковые номера следующими способами.Therefore, backup assigns sequence numbers in the following ways:

    • Последовательные семейства носителей в наборе носителей.Sequential media families within a media set

      Семейства носителей в наборе носителей последовательно нумеруются в соответствие с их позицией в наборе носителей.Within a media set, the media families are numbered sequentially according to their position in the media set. Номер семейства носителей записывается в столбец family_sequence_number таблицы backupmediafamily .The media-family number is recorded in the family_sequence_number column of the backupmediafamily table.

    • Физический носитель в семействе носителей.Physical media within a media family

      Порядковый номер носителя указывает на место физического носителя в семействе носителей.A media sequence number indicates the order of the physical media within a media family. Порядковый номер начального носителя резервной копии равен 1.The sequence number is 1 for the initial backup media. Он обозначен цифрой 1, второй (первый дополнительный носитель) обозначен цифрой 2 и т.д.This is tagged with 1; the second (the first continuation tape) is tagged with 2; and so on. При восстановлении из резервного набора данных наличие порядковых номеров носителей позволяет гарантировать, что оператор, выполняющий восстановление, монтирует носители в правильном порядке.When the backup set is restored, the media sequence numbers make sure that the operator restoring the backup mounts the correct media in the correct order.

    Несколько устройств Multiple Devices

    При использовании нескольких ленточных накопителей или дисковых файлов необходимо помнить о следующем.When you use multiple tape drives or disk files, the following considerations apply:

    • При резервном копировании.For backup:

      Весь набор носителей, созданный операцией резервного копирования, должен целиком использоваться всеми последующими операциями резервного копирования.The complete media set that is created by a backup operation must be used by all subsequent backup operations. Например, если в созданном наборе носителей используется два накопителя на магнитной ленте для резервного копирования, то все последующие операции резервного копирования, включающие этот набор носителей, должны использовать два устройства резервного копирования.For example, if a media set was created by using two tape backup devices, all subsequent backup operations that involve the same media set must use two backup devices.

    • При восстановлении.For restore:

      При любом восстановлении из резервных копий на диске или восстановлении «в сети», необходимо последовательно подключить все семейства носителей.For any restore from disk backups and for any online restore, all the all media families must be concurrently mounted. При восстановлении вне сети с ленточных устройств резервного копирования можно обработать семейства носителей с меньшего количества устройств резервного копирования.For an offline restore from tape backups, you can process the media families from fewer backup devices. Каждое семейство носителей полностью обрабатывается перед тем, как начинается обработка другого.Each media family must be processed completely before starting to process another media family. Семейства носителей всегда обрабатываются параллельно, если происходит восстановление более чем с одного устройства.Media families are always processed in parallel, unless they are being restored with a single device.
