Spamassassin: подключаем к Exim на CentOS

By мережевий хробачок™ | 07/15/2014
0 Comments
 
Rate this (1 Vote)

Устанавливаем пакет:

# yum -y install spamassassin

ВАЖНО: требуется подключенный репозиторий rpmforge-extras. При установке возможны проблемы типа Error: Package: perl-IO-Compress-2.052-1.el6.rfx.noarch.

После установки рабочий каталог Spamassassin выглядит так:

Настройка Exim описана в статье CentOS: установка и настройка Dovecot 2 + Exim + ClamAV + Postfixadmin.

По-умолчанию Spamassassin создаёт каталог для базы Bayes в домашней директории пользователя, который его запускает:

view source
1# ls -l /root/.spamassassin
2total 284
3-rw------- 1 root root 12288 Jul 14 08:06 bayes_seen
4-rw------- 1 root root 331776 Jul 14 08:06 bayes_toks

Т.к. нам это нежелательно и проще хранить всё в одном месте – то есть два варианта.

Более простой – хранить базы в заданной в конфигурации директории. Второй – в базе MySQL.

Пока выберем первый.

Создаём файлы для Bayes базы:

view source
1# mkdir /etc/mail/spamassassin/bayes/
2# touch /etc/mail/spamassassin/bayes/bayes_

Дописываем в основной конфиг-файл /etc/mail/spamassassin/local.cf:

view source
01# количество "очков спама", что бы письмо было помечено как спам
02required_hits 5
03 
04# что следует делать с оригинальным сообщением, если оно будет признано спамом
05# оставить как есть - 0
06# прикрепить к отчету как вложение - 1
07# добавить к отчету как текст - 2
08report_safe 0
09 
10# Добавлять метку *****SPAM***** к спам сообщениям
11rewrite_header Subject *****SPAM*****
12 
13# Указываем локальную сеть как доверенную, т.е.
14# исходящие письма не проверяем
15# в данном случае не доверяем никому
16# trusted_networks
17 
18# Указываем кол-во баллов, при котором сообщение считается спамом
19# (по умолчанию: 5.0)
20required_score 5.0
21 
22# Использовать фильтр Bayesian (по умолчанию: 1)
23use_bayes 1
24 
25# Когда производится обучение по bayes, эта информация падает в
26# журнал
27bayes_learn_to_journal 1
28 
29# путь к префиксу файлов bayes-базы - НЕ директории
30bayes_path /etc/mail/spamassassin/bayes/bayes
31# Указываем сколько чистых писем и писем со спамом должно
32# накопится в Bayesian базе прежде чем демон будет пользоваться
33# этой базой (по умолчания - 200 не спама и спама)
34bayes_min_ham_num 1
35bayes_min_spam_num 1
36 
37# Bayesian авто обучение (default: 1)
38# письма распознанные как 100% спам или 100% не спам
39# добавляются в базу
40bayes_auto_learn 1
41 
42# не анализировать письма, содержащие указанный тэг в заголовке
43bayes-ignore-header <тэг_заголовка>
44 
45# не подвергать анализу письма от указанного адресата
46bayes_ignore_from <адрес>
47 
48# не анализировать сообщения, предназначенные указанному пользователю
49bayes_ignore_to <адрес>
50 
51# Не использовать сетевые проверки (т.к. делаем это в MTA)
52skip_rbl_checks 0
53 
54# Использовать Razor
55use_razor2 0
56 
57# Не использовать Pyzor
58use_pyzor 0
59 
60# Указываем допустимые языки и локали сообщений
61ok_locales en ru
62 
63# Автоматический белый список выключаем, поскольку в него
64# зачастую попадают спамеры.
65# use_auto_whitelist 0
66# auto_whitelist_path /etc/mail/spamassassin/auto-whitelist
67# auto_whitelist_file_mode 0666
68 
69# не проверять почту от указанных адресов;
70# можно указать шаблон - символ «*» обозначает любое количество символов,
71# «?» – любой один символ
72# whitelist_from <адрес_или_шаблон>
73 
74# аналогично «белому» списку, но теперь отправитель априори рассматривается
75# как спамер, и все письма от него помечаются как спам без дальнейшего анализа
76# blacklist_from <адрес_или_шаблон>
77 
78# все сообщения на указанный адрес будут передаваться без фильтрации;
79# можно использовать, если владелец адреса желает получать всю почту, адресованную ему;
80# также рекомендуется устанавливать этот параметр для адреса abuse,
81# чтобы даже в случае неправильной настройки фильтра с вами можно было связаться.
82# whitelist_to <адрес_или_шаблон>
83 
84# все сообщения на указанный адрес будут рассматриваться как спам;
85# например, таким образом можно организовать «прививки» для статистического анализатора,
86# создав специальный адрес для спама и максимально «засветив» в интернете
87# (при этом должно быть включено автообучение)
88# blacklist_to <адрес_или_шаблон>

Проверяем синтаксис файла:

view source
1# spamassassin --lint

Запускаем обновление баз для Spamassassin:

view source
1# sa-update -v
2Update available for channel updates.spamassassin.org
3Update was available, and was downloaded and installed successfully

Запускаем и добавляем в автозапуск:

view source
1# service spamassassin start
2Starting spamd: [ OK ]
view source
1# chkconfig spamassassin on

Проверяем порт:

view source
1# netstat -anp | grep spam
2tcp 0 0 127.0.0.1:783 0.0.0.0:* LISTEN 23011/spamd.pid

Переходим к Exim.

В файле /etc/exim/exim.conf добавляем:

view source
1spamd_address = 127.0.0.1 783

Дальше, в блок acl_check_data: добавляем:

view source
01# указываем адрес на который пересылать сообщения
02# определенные как спам, на случай если это ошибка.
03warn message = X-Redirect-To: spam@domain.org.ua
04 hosts = !+relay_from_hosts
05 spam = nobody
06 
07# пишем в заголовки баллы за спам
08warn spam = nobody:true
09hosts = !+relay_from_hosts
10message = X-Spam-Level: $spam_bar
11 
12# пишем в заголовки отчет spamassassin'a
13warn spam = nobody:true
14 hosts = !+relay_from_hosts
15 condition = ${if >{$spam_score_int}{29}{1}{0}}
16 message = X-Spam-Status: $spam_report
17 
18# если письмо набрало больше 10 баллов, то
19# не принимаем его -устанавливаем deny
20# что бы переслать на ящик spam@domain.org.ua - ставим accept
21accept message = Message scored $spam_score spam points
22 spam = nobody:true
23 condition = ${if >{$spam_score_int}{100}{yes}{no}}
24 # и записываем сообщение в лог
25 logwrite = Email from $sender_address with $spam_score spam points redirected to spam@domain.org.ua by SpamAssassin

В блок begin routers добавляем пересылку писем:

view source
1begin routers
2 
3# роутер пересылающий спам в ящик spam@domain.ru
4scan_redirect:
5 driver = redirect
6 condition = ${if def:h_X-Redirect-To: {1}{0}}
7 headers_add = X-Original-Recipient: $local_part@$domain
8 data = $h_X-Redirect-To:
9 headers_remove = X-Redirect-To

Проверяем конфиг:

view source
1# exim -bV

Перезапускаем Exim:

view source
1# service exim restart
2Shutting down exim: [ OK ]
3Starting exim: [ OK ]

Пробуем отправить письмо с текстом:

XJS*C4JDBQADN1.NSBN3*2IDNEN*GTUBE-STANDARD-ANTI-UBE-TEST-EMAIL*C.34X#sthash.oh8083pv.dpuf

На любой ящик сервера, и смотрим лог /var/log/exim/main.log:

view source
12014-07-14 22:06:44 1X6laG-0008Nh-Ff Email from setevoy@domain.ua with 1001.8 spam points redirected to spam@domain.org.ua by SpamAssassin

Spamassasin работает.

Когда накопится какое-то количество спама в определённой папке (например – Thunderbird имеет собственный антиспам, и складывает спам в отдельную папку + можно самому помечать письма как спам) – можно запустить утилиту sa-learn для обучения:

view source
1# sa-learn --spam /var/vmail/domain.kiev.ua/1th@domain.kiev.ua/.Junk/
2Learned tokens from 66 message(s) (67 message(s) examined)

И для обучения “хорошим” письмам:

view source
1# sa-learn --ham /var/vmail/domain.kiev.ua/1th@domain.kiev.ua/cur/

Можно добавить вызов в cron:

view source
1# crontab -e
view source
1# crontab -l
200 */12 * * * /usr/bin/sa-learn --spam /var/vmail/domain.kiev.ua/1th@domain.kiev.ua/.Junk/

Проверку “хорошей” почте в cron пока не добавляем, т.к. ещё много спама будет попадать в новую почту.

И проверим базу bayes:

view source
1# sa-learls -l bayes/
2total 284
3-rw-r--r-- 1 root root 0 Jul 14 08:21 bayes_
4-rw------- 1 root root 12288 Jul 14 08:24 bayes__seen
5-rw------- 1 root root 331776 Jul 14 08:24 bayes__toks
